## ⚠️ ВАЖНО для продакшена:

### 1. НЕ храните продакшен ключи в клиентском коде!

**Текущая реализация с ключами в `config/tickets.json` подходит ТОЛЬКО для:**
- ✅ Тестирования
- ✅ Разработки
- ✅ Демо-версий

### 2. Для продакшена ОБЯЗАТЕЛЬНО:

1. **Создайте бэкенд API** (см. `backend-example.js`)
2. **Храните ключи на сервере** в переменных окружения:
   ```bash
   YOOKASSA_SHOP_ID=your_production_shop_id
   YOOKASSA_SECRET_KEY=your_production_secret_key
   ```
3. **Все платежи должны проходить через ваш бэкенд**
4. **Никогда не отправляйте секретные ключи на клиент**

### 3. Что делать сейчас:

Для тестирования текущая настройка подходит. Но перед запуском в продакшен:

1. Удалите ключи из `config/tickets.json`
2. Создайте бэкенд API
3. Обновите код в `src/pages/Payment.jsx` для работы с вашим API
4. Используйте переменные окружения на сервере

### 4. Проверка безопасности:

- ✅ Тестовые ключи можно хранить в репозитории
- ❌ Продакшен ключи НИКОГДА не должны быть в репозитории
- ✅ Используйте `.env` файлы (они в `.gitignore`)
- ✅ Используйте секреты в CI/CD системах
